HTML Accessibility Best Practices

1. Use Semantic HTML

  • Use headings (<h1> to <h6>) in order. Don’t skip levels.
  • Use <nav>, <main>, <footer>, <header>, <section>, <article> for structure.

2. Add Alt Text to Images

<img src="cat.jpg" alt="A brown cat sitting on a sofa">
  • Describe the image’s content and function.
  • If the image is decorative, use alt="".

3. Label Form Elements

<label for="email">Email:</label>
<input type="email" id="email" name="email" required>
  • Always use <label> for inputs, checkboxes, and radios.
  • Use aria-label or aria-labelledby for custom controls.

4. Use Sufficient Color Contrast

  • Text should be easy to read against its background.
  • Don’t use color alone to convey meaning.

5. Keyboard Navigation

  • All interactive elements (links, buttons, forms) should be accessible with Tab/Shift+Tab.
  • Use tabindex to control tab order if needed.

6. ARIA Attributes (Intro)

  • Use role, aria-label, aria-labelledby, and aria-describedby for custom widgets.
  • Only use ARIA when native HTML can’t do the job.
